Git技巧之搭建服务器仓库

2015-08-23
技术研发

今天把博客从wordpress迁移到了Hexo,部署非常简单,Hexo也提供了github部署功能.但是考虑到一些扩展,会用到在主机上搭建了一个git服务,大概步骤如下:

一、安装Git

  • 方式一:
    $ sudo apt-get install git

  • 方式二:

    • Git支持工具:

      $ apt-get install libcurl4-gnutls-dev libexpat1-dev gettext \ libz-dev libssl-dev

    • 官网下载源码:http://git-scm.com/download

    • 解压编译安装:

      $ tar -zxf git-1.7.2.2.tar.gz

      $ cd git-1.7.2.2

      $ make prefix=/usr/local all

      $ sudo make prefix=/usr/local install

  • 测试:
    $ git --version

二、创建系统用户

  • 创建Git用户名:

    $ sudo adduser git

  • 设置Git密码:

    $ sudo passwd git

三、创建客户端证书

  • 生成开发机公钥:

    $ ssh-keygen -t rsa

  • 上传公钥并添加:

    $ scp ~/.ssh/id_rsa.pub git@localhost:/tmp //localhost更换成主机地址

    $ cat /tmp/id_rsa.pub >> ~/.ssh/authorized_keys

四、创建服务器仓库

  • 初始化版本仓库:

    $ git init --bare project.git

  • 设置用户和用户组权限(如果git用户无权限访问):

    $ chown -R git project.git *

    $ chgrp -R git project.git *

五、禁用shell登录权限(如果有安全考虑)

  • 找到/etc/passwd将:

    git:x:1000:1000::/home/git:/bin/bash

    改为

    git:x:1000:1000::/home/git:/bin/git-shell

遇到的问题

  • 权限问题,当用root用户去初始化的project.git项目时,务必将目录的所有权限提供给git用户.若还要建立其他git用户,直接建立个用户组git.

    $ chown -R git project.git *

    $ chgrp -R git project.git *

其他

  • 克隆远程仓库:

    git clone git@server:/srv/sample.git

  • 管理公钥:

    • 方式一:把公钥收集起来放到服务器的/home/git/.ssh/authorized_keys文件即可
    • 方式二:使用Gitosis

  • 管理权限:通过Gitolite
